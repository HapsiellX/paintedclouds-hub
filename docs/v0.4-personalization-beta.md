# StefARR V0.4 personalization beta

V0.4 adds the authenticated **For you** feed and one saved list across movies,
series, music, e-books, and audiobooks. Personalization is enabled by default
and can be disabled or erased from the feed preferences.

## Privacy model

Ranking uses only the current user's explicit likes, hidden titles, saved items,
Hub requests, chosen genres/languages, and the existing local library. Playback
history, other users' signals, LLMs, advertising identifiers, and third-party
tracking are not used. Operators can see aggregate provider health and error
codes, never individual preferences. Feedback never starts a request or a
download.

The identity and three explicit boolean states are stored in `hub_user_signal`.
A minimal title/image snapshot is retained only so the saved list can be shown
without another provider call. `DELETE /api/v1/hub/personalization/data`
removes all signals and preferences and leaves personalization disabled.

## API

- `GET|PUT /api/v1/hub/personalization/profile`
- `PUT /api/v1/hub/personalization/items` (idempotent `liked`, `hidden`, `saved`)
- `DELETE /api/v1/hub/personalization/data`
- `GET /api/v1/hub/recommendations?mediaType=all&pageSize=24&cursor=...`
- `GET /api/v1/hub/saved?pageSize=24&cursor=...`
- `GET /api/v1/hub/providers/status` (administrators only)

Recommendation reasons are stable codes (`LIKED_SIMILAR`, `SAVED_SIMILAR`,
`REQUEST_SIMILAR`, `PREFERRED_GENRE`, `PREFERRED_LANGUAGE`,
`LIBRARY_SIMILAR`, `POPULAR_COLD_START`, `REDISCOVER`) and are translated in
the client. Cursors are opaque and invalid cursors return HTTP 400.

## Ranking and provider behavior

Hidden items are excluded. Likes have the strongest weight, followed by saved
items, own requests, language/genre preferences, and library similarity. The
mixed row is round-robin diversified by media kind. Ties use a stable identity
hash, so equal inputs give equal results. Available media is shown only in the
rediscovery row.

Metadata calls retain strict timeouts and provider rate limits. Three consecutive
failures open a provider circuit for five minutes. Successful candidates are
cached for one hour and remain available as a stale fallback when every provider
is unavailable. The admin Hub view reports the last successful sync, circuit
state, and a bounded history of neutral error codes.

## Upgrade and rollback

Back up the database and configuration directory before upgrading. The
`1785001000000-AddHubPersonalization` migrations add two tables with cascading
user deletion and do not modify V0.3 tables; they support SQLite and PostgreSQL.
Start `v0.4.0-beta.1`, wait for migrations, check `/api/v1/status`, then verify
the For-you feed and provider cards as an administrator.

Downgrade is supported only by restoring the V0.3 database/configuration backup.
Do not run the migration `down()` path on a live installation because it deletes
all personalization data.

## Beta limitations

Recommendations depend on metadata-provider coverage and may temporarily use
cached candidates. There is no collaborative filtering, playback tracking,
social activity, automatic request creation, or new download backend. Radarr,
Sonarr, Lidarr, and LazyLibrarian remain authoritative for acquisition.
