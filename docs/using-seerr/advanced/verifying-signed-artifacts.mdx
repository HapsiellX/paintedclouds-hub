# Verifying signed PaintedClouds Hub images

Every published release promotes an already tested multi-architecture candidate
by immutable digest. The release pipeline creates a keyless Sigstore signature,
a CycloneDX SBOM and an SBOM attestation.

Use the exact image and digest printed in the matching
[GitHub release](https://github.com/HapsiellX/paintedclouds-hub/releases). The
examples below intentionally do not use mutable tags.

```bash
IMAGE='ghcr.io/hapsiellx/paintedclouds-hub@sha256:RELEASE_DIGEST'

cosign verify "$IMAGE" \
  --certificate-identity 'https://github.com/HapsiellX/paintedclouds-hub/.github/workflows/release-candidate.yml@refs/heads/main' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com'
```

Verify the attached CycloneDX attestation with the same workflow identity:

```bash
cosign verify-attestation "$IMAGE" \
  --type cyclonedx \
  --certificate-identity 'https://github.com/HapsiellX/paintedclouds-hub/.github/workflows/release-candidate.yml@refs/heads/main' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com'
```

To inspect the registry digest independently:

```bash
docker buildx imagetools inspect \
  ghcr.io/hapsiellx/paintedclouds-hub:0.3.0 \
  --format '{{json .Manifest.Digest}}'
```

The digest must exactly match the release notes before you deploy it. A valid
signature proves which GitHub workflow produced the artifact; it does not
replace reading the migration notes, scanning the image or creating a backup.
