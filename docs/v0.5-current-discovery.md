# PaintedClouds Hub V0.5 current discovery

V0.5 replaces the fixed music genre shelves with user-controlled music
sources and makes movie, series, and music recommendations time-sensitive.

## Music sources

Each user can independently configure up to 20 MusicBrainz genres/tags and 20
artists or groups. Artist choices use canonical MusicBrainz IDs so namesakes do
not get mixed. Genre values and artist choices are stored in the existing local
`hub_user_profile`; they are never shared with other users.

The music start page creates one current-release shelf per selected genre and
artist. It searches MusicBrainz release groups from the previous six months and
includes albums, EPs, and singles. When the user has no music source configured,
the page shows a general **Current music** shelf instead of fixed genres.

## Current movies and streaming series

Movie discovery now starts with separate **Trending now** and **New popular
movies** shelves. Series discovery uses episode air dates from the preceding
120 days, so a current season of an established show remains eligible. It
offers separate shelves for popular streaming series, acclaimed current series,
and current popular anime. The same current-window logic feeds personalized
recommendations.

Titles must be available as a flatrate stream from major providers in the
user's configured watch region. Provider IDs, including Crunchyroll where
available, are resolved from TMDB at runtime; no fragile static IDs are stored.
Linear television and catch-up providers are deliberately not used as a
fallback. TMDB popularity and ratings are described as "popular" or
"acclaimed", not as exact cross-service viewing figures.

Personalized TMDB recommendation/similar results are also limited to the recent
window. Existing media remains confined to the rediscovery row.

## Search layout

Routes with a dedicated contextual search (`/`, `/hub`, `/discover/music`, and
`/discover/books`) no longer render a second global search input above it. The
toolbar, mobile navigation, and user menu remain available.

## Navigation ownership

V0.5.0-beta.3 removes the duplicate **Hub** entry from desktop and mobile
navigation. **Discover** remains the single cross-media entry point and `/hub`
stays available as a compatibility alias. **Requests** now opens the dedicated
request-management page instead of jumping to a section inside Discover.
Personal recommendations and the shared saved list are available in the mobile
navigation as well, with visible labels and accessible current-page states.

## Upgrade

The `1785002000000-AddHubMusicPreferences` migration adds nullable
`musicGenres` and `musicArtists` JSON columns to `hub_user_profile` for SQLite
and PostgreSQL. Back up the V0.4 database before upgrading to
`v0.5.0-beta.3`. A downgrade remains backup/restore only.
