# Docker

StefARR by PaintedClouds publishes multi-architecture images only to GitHub Container
Registry. Do not use Seerr images, Docker Hub mirrors, `latest`, or an
unversioned development image for a StefARR by PaintedClouds installation.

## Before you start

1. Open the matching [GitHub release](https://github.com/HapsiellX/paintedclouds-hub/releases).
2. Read its supported upgrade path and create a database plus configuration
   backup.
3. Copy the exact `ghcr.io/hapsiellx/paintedclouds-hub@sha256:…` reference from
   those release notes.
4. Verify the digest, signature and SBOM attestation before starting it.

## Docker CLI

```bash
install -d -m 0750 /path/to/paintedclouds-hub/config
chown -R 1000:1000 /path/to/paintedclouds-hub/config

docker run -d \
  --name paintedclouds-hub \
  --init \
  --restart unless-stopped \
  -e TZ=Europe/Berlin \
  -p 5055:5055 \
  -v /path/to/paintedclouds-hub/config:/app/config \
  ghcr.io/hapsiellx/paintedclouds-hub@sha256:RELEASE_DIGEST
```

Replace `RELEASE_DIGEST` with the full digest published for the version you are
installing. The application runs as UID 1000 and listens on port 5055.

## Docker Compose

```yaml
services:
  paintedclouds-hub:
    image: ghcr.io/hapsiellx/paintedclouds-hub@sha256:RELEASE_DIGEST
    init: true
    restart: unless-stopped
    environment:
      TZ: Europe/Berlin
    ports:
      - 5055:5055
    volumes:
      - /path/to/paintedclouds-hub/config:/app/config
    healthcheck:
      test: [CMD, wget, --spider, -q, http://127.0.0.1:5055/api/v1/status]
      start_period: 60s
      timeout: 10s
      interval: 30s
      retries: 5
```

Use a persistent PostgreSQL service and the documented `DB_*` variables when
you prefer PostgreSQL over the default SQLite database. Back up both the
database and `/app/config`; encrypted Hub credentials also require the matching
`hub-secrets.key`.

## Upgrading

Stop the application, take a consistent backup, change only the image reference
to the new release digest, and start one application instance. Validate login,
search, requests, integration tests and reconciliation before removing the
backup. Database downgrades are unsupported; rollback requires restoring the
matching old image, database and configuration backup together.
