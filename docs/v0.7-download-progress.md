# StefARR V0.7 download progress

V0.7 makes active Radarr and Sonarr work visible in the unified **Requests**
view instead of hiding progress inside detail-page tooltips.

## What users see

- A download queue with an overall progress bar and one card per requested
  movie or series.
- Byte-weighted progress for each movie and series download.
- Separate progress rows for individual episode downloads and season packs,
  including `SxxExx` episode labels, status and remaining time when supplied by
  Sonarr.
- The same compact progress bar directly on the matching request.
- A clear empty queue when Radarr and Sonarr have no active work.

The view refreshes every 30 seconds. StefARR's existing download sync polls the
configured Radarr and Sonarr instances every minute, so displayed values may
trail the download client briefly.

## Privacy and authorization

The queue follows the existing request visibility rules. Users without the
**View requests** or **Manage requests** permissions receive progress only for
their own requests. Authorized request viewers receive the shared queue.

Raw download-client identifiers and release filenames are never returned by
the unified activity API. Only the requested title, sanitized state, progress,
byte totals, remaining time and episode numbers are exposed.

## Scope

V0.7 progress bars use Radarr and Sonarr's established queue APIs. Lidarr and
LazyLibrarian continue to expose their normalized request lifecycle states;
their queue formats are not presented as byte-level progress in this release.

No database migration is required. Upgrades from V0.6 use the existing direct
upgrade path. Downgrades remain backup/restore only.
